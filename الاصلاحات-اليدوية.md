# إصلاحات يدوية إضافية (بعد إصلاح `tailwind.config.js`)

إصلاح ملف `tailwind.config.js` وحده يُعيد ~90% من الألوان. تبقّت أخطاء يدوية دقيقة أدناه.

---

## 1) نصوص أصناف مكسورة: كود لون خام مكتوب مكان اسم الصنف

استخدام كود `#RRGGBB` مباشرة كـ className **لا يعمل** إطلاقاً (ليس صنف Tailwind) → العنصر بلا لون.

| الملف | السطر | الحالي (خطأ) | الصحيح |
|---|---|---|---|
| `src/components/sheets/OrderDetailSheet.jsx` | 26 | `badge: '#F4F7F9 text-text-secondary'` | `badge: 'bg-background text-text-secondary'` |
| `src/components/sheets/OrderDetailSheet.jsx` | 27 | `badge: '#F4F7F9 text-text-tertiary'` | `badge: 'bg-background text-text-tertiary'` |
| `src/components/sheets/PinEntrySheet.jsx` | 64 | `... : '#E4EAEE'` | `... : 'bg-border'` |
| `src/components/ui/CalendarView.jsx` | 154 | `STATUS_CONFIG[status]?.color \|\| '#93A4AE'` | `... \|\| 'bg-disabled'` |

## 2) حدود بألوان خارج الهوية (arbitrary hex)

تعمل تقنياً لكنها تكسر انضباط الهوية. استبدلها برموز النظام:

| الملف | الحالي | الأفضل |
|---|---|---|
| `src/pages/OrdersPage.jsx:14` | `bar: 'border-[#f5a623]'` | `bar: 'border-withdrawal-500'` |
| `src/pages/OrdersPage.jsx:16` | `bar: 'border-[#b0b6c3]'` | `bar: 'border-divider'` |

## 3) نظام الثيم المعطّل (سبب رئيسي لشعور "لم يُطبَّق كل شيء")

`src/utils/theme.js`:
- `DEFAULT_PRIMARY = '#1F6FE8'` (أزرق سامسونج) — لا يمتّ للهوية بصلة.
- `applyTheme()` يكتب متغيّرات `--color-primary-*` على `:root`، **لكن `tailwind.config.js` لا يقرأ أي `var(--color-primary)`** (يستخدم قيم hex ثابتة). فالنتيجة أن مبدّل الألوان في الإعدادات **لا يفعل شيئاً**، وقائمة `THEME_PRESETS` كلها ألوان One-UI تخالف الهوية.

**الخيار الموصى به (هوية ثابتة لنظام مالي):** احذف مبدّل الثيم من `SettingsPage`، واحذف `applyThemeFromDB()` من `App.jsx`، فالكحلي `#023852` هو لون الهوية الثابت.

**إن أردت الإبقاء على تبديل اللون الأساسي فقط:** اربط `primary` في الإعداد بالمتغيّرات:
```js
primary: {
  50:'var(--color-primary-50)', 100:'var(--color-primary-100)', /* ... */ 900:'var(--color-primary-900)',
  DEFAULT:'var(--color-primary-500)'
}
```
واضبط `DEFAULT_PRIMARY = '#023852'` واستبدل الـ presets بدرجات معتمدة، وعرّف قيم `:root` الافتراضية في `index.css` حتى لا تختفي الألوان قبل تحميل الثيم.

---

## 4) لون النص الأساسي داكن جداً (يفاقم شعور الإظلام)

`src/styles/index.css` → `body { @apply ... text-ink ... }` و`ink.DEFAULT = #023852` (كحلي داكن).
أي نص بلا لون صريح يرث هذا الكحلي الداكن. بعد إصلاح التدرّجات ستُحل معظم الحالات، لكن يُنصح بجعل النص الأساسي أهدأ:

```css
body { @apply bg-background text-ink-strong font-sans antialiased; } /* #2C3E47 بدل #023852 */
```
وابقِ `#023852` للعناوين والأرقام المهمة فقط.

## 5) تنظيف

- `income_legacy` في الإعداد غير مستخدم — احذفه.
- تكرار `withdraw` و`withdrawal` مربك — وحّدهما تدريجياً على `withdrawal`.
- تعليقات "One UI style" في `Icon.jsx` وتعليق مضلّل في `theme.js` ("The Tailwind config references var(--color-primary)") — حدّثها.
- `getGreeting()` في `date.js`: الفروع الثلاثة المسائية كلها تعيد "مساء الخير" (غير ضار).
