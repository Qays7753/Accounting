## الأزرار والروابط المُفحوصة

### ✅ تعمل بشكل صحيح
- جميع أزرار FAB (إضافة معاملة/طلب) → تفتح الأوراق المناسبة
- تبويبات BottomNav (الرئيسية/المالية/الطلبات/POS/المخزون/الإعدادات) → تتنقل صحيحاً
- أزرار الحفظ في كل الأوراق → تعمل مع useSubmitGuard
- أزرار الحذف → تطلب تأكيد
- أزرار WhatsApp → تفتح wa.me أو Web Share
- أزرار الاتصال → tel: links
- مبدّل الوضع (يومي/مدير/مستثمر) → يبدّل فوراً
- أزرار المزامنة السحابية → تفتح OAuth / تزامن / قطع
- زر PDF في InvestorDashboard → يُصدّر بشكل صحيح (dynamic import)
- تبديل Quick POS → يُظهر/يُخفي التبويب
- تبديل المخزون → يُظهر/يُخفي التبويب
- التسوية الأسبوعية → تعمل مع حساب الفرق

### 🔧 أُصلِحت
1. **زر «نوع النشاط» في الإعدادات:** كان يُظهر `alert()` غير مفيد. أُصلِح ليفتح BottomSheet لاختيار نوع النشاط (جاهزة/مُصنَّعة) مع حفظ في Dexie.

### ⚠️ ملاحظات (لا تعطل لكن تحتاج تحسين مستقبلي)
1. **11 استدعاء `alert()`** بدلاً من Snackbar/Toast — مقبول للآن لكن يُفضَّل استبدالها بنظام toast موحَّد.
2. **عدم وجود toast لتأكيد نجاح المزامنة** — المستخدم لا يعرف إذا تمت المزامنة بنجاح إلا بفحص «آخر مزامنة».

## الفرص المُكتشَفة

1. **تأكيد بصري عند الحفظ:** عند حفظ معاملة/طلب، الورقة تُغلق فوراً بدون تأكيد بصري. إضافة Snackbar «تم الحفظ» يحسّن الثقة.
2. **عدّاد تنازلي للتسوية الأسبوعية:** بدلاً من إظهار البطاقة فقط، يمكن إظهار «مرّ X يوم منذ آخر تحديث» لخلق إحساس بالإلحاح.
3. **اهتزاز نجاح عند تصدير PDF:** حالياً `hapticSuccess` يُستدعى لكن الاهتزاز قد لا يكون كافياً للتأكيد على عملية طويلة (توليد PDF).
4. **تحديث «آخر مزامنة» بصرياً:** عند نجاح المزامنة، تحديث الوقت فوراً في بطاقة الحالة بدل انتظار re-render.
