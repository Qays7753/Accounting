## نظرة عامة: الطبقات الثلاث

التطبيق مبني على مبدأ «طبقات تراكمية» — كل طبقة تضيف تعقيداً وذكاءً فوق سابقتها. المستخدم يبدأ في Layer 1 و«يرفع المستوى» عندما يكون جاهزاً.

---

## Layer 1: اليومي (Daily) — «احسب وفهّم»

### المستخدم المستهدف
صاحب محل صغير (بقالة، كافيه، محل عصائر) يدير المحل بنفسه. يريد ببساطة أن يعرف: كم بالدرج؟ كم ربحت اليوم؟ كم عليّ/لي؟

### المصطلحات
- لغة الشارع الأردنية: «حق المحل»، «حق التاجر»، «كم قبضت»، «كم صرفت»
- أرقام بسيطة، بطاقات محادثة، لا جداول معقدة

### الشاشات والرحلة

**1. الرئيسية (Home):**
- شعار + تحية (صباح/مساء الخير)
- بطاقة «الدرج الصادق»: بالدرج (صغير) + إلك منها الحرة (كبير تِراكوتّا)
- بطاقتا الجرتين: حق المحل (تِراكوتّا) + حق التاجر (أخضر)
- بطاقات اليوم: قبض اليوم / صرف اليوم
- الطلبات القادمة (٣ طلبات)
- ملخص الشهر (إن كان مُفعّلاً)
- تذكير الإقفال اليومي (بعد الوقت المحدد)
- تذكير المخزون الأسبوعي (إن كان مُفعّلاً)
- بطاقات SWOT التشخيصية (إن وُجدت رؤى)

**2. المالية (Finance):**
- شريط علوي: عنوان + بحث (أيقونة تتوسّع)
- بطاقة صافي الشهر + قبض/صرف
- segmented control: الكل / قبض / صرف / سحب
- قائمة المعاملات مجمّعة باليوم مع الصافي
- بطاقات وصول سريع: الديون / التقارير

**3. الطلبات (Orders):**
- شريط علوي: عنوان + إضافة
- segmented: قائمة / تقويم
- تبويبات حالة مع عدّادات
- بطاقة طلب: اسم الزبون + نوع + حالة + مبلغ + أزرار اتصال/واتساب

**4. البيع السريع (POS) — اختياري:**
- شبكة منتجات + سلة مصغّرة
- دفع نقداً / أجل
- لا خصم مخزون تلقائي (Layer 1 = منفصل)

**5. المخزون — اختياري:**
- قائمة أصناف بالعدد/بالعادة
- تسجيل شراء + تذكير أسبوعي للتسوية
- لا خصم تلقائي عند البيع

**6. الإعدادات:**
- المزامنة السحابية (Google Drive)
- بيانات النشاط (اسم المحل + نوع النشاط)
- النسخ الاحتياطي اليدوي (Plan B)
- وضع التطبيق (مبدّل ٣ أوضاع)
- الأمان / العرض / التشغيل

**شريط التنقّل السفلي (5-6 تبويبات):**
الرئيسية / المالية / الطلبات / [POS اختياري] / [المخزون اختياري] / الإعدادات

## Layer 2: المدير (Manager) — «حلّل وحسّن»

### المستخدم المستهدف
مدير يريد فهماً أعمق: هامش الربح، دوران المخزون، مصادر الهدر، أداء الموظفين.

### المصطلحات
- محاسبية رسمية: «رأس المال المستثمر»، «الأرباح المحتجزة»، «صافي التدفق النقدي»
- جداول بيانات + أرقام مفصّلة

### ما يتغير عن Layer 1

**الإضافات:**
1. **مخزون تنبؤي:** النظام يحسب تلقائياً متى تطلب كل صنف بناءً على إيقاع المشتريات
2. **BOM (وصفة):** بيع كوب شاي يخصم تلقائياً: ١ كوب + ٢غ شاي + ٥مل حليب
3. **خصم تلقائي عند البيع:** POS يخصم المخزون المرتبط بالمنتج
4. **تقارير تفصيلية:** هامش لكل منتج، مصادر الهدر، أداء الموظفين
5. **فلاتر متقدمة:** حسب الفئة / الموظف / النوع

**ما يبقى كما هو:**
- نفس شريط التنقّل + نفس الهيدر + نفس الألوان
- الفرق: المصطلحات تتغيّر (simple → pro) + تظهر بيانات إضافية
- تبويب المخزون يصبح إلزامياً (لا يمكن إخفاؤه)

**رحلة المستخدم النموذجية:**
1. يفتح التطبيق → يرى Dashboard بنفس الشكل لكن بمصطلحات احترافية
2. يذهب للمخزون → يرى تنبؤات الطلب + تحذيرات نفاد المخزون
3. يسجّل بيعاً → النظام يخصم المخزون تلقائياً
4. يفتح التقارير → يرى هامش كل منتج + تحليل الهدر
5. يرى بطاقة SWOT: «هامش كوب الشاي ١٥٪ — راجع التسعير»

### التغييرات التقنية المطلوبة
- `linked_item_id` في `quick_products`
- جدول `bom_components`
- جدول `purchase_records` (بدل array في items)
- `auto_deduct: boolean` في items
- `item_type: 'raw' | 'finished'` في items

## Layer 3: المستثمر (Investor) — «اعرض وقرّر»

### المستخدم المستهدف
مستثمر / شريك / بنك يريد رؤية شاملة مختصرة: هل المحل مربح؟ ما هي قيمة الأصول؟ كم يوم صمود؟

### المصطلحات
- محاسبية رسمية (نفس Layer 2)
- أرقام ضخمة Mono، بطاقات بطل تِراكوتّا، خلفية بيضاء ناصعة

### ما يتغيّر جذرياً

**التخطيط:**
- يختفي شريط التنقّل السفلي + FAB
- تظهر لوحة تنفيذية للقراءة فقط (InvestorDashboard)
- خلفية `#FFFFFF` (ناصع) بدل عاجي `#FAF9F5`
- كتل تِراكوتّا جريئة كأسطح بطل

**المحتوى:**
1. **هرم المؤشرات (10 KPIs):**
   - صف بطل: أيام الصمود + هامش صافي الربح (تِراكوتّا، ٣٤px Mono)
   - شبكة ٢×٢: هامش إجمالي، ROI، نسبة مصروفات، متوسط طلب
   - شبكة ٢×٢: دوران مخزون، DSO، دوران نقد، نسبة سيولة

2. **قائمة الدخل الشلالية:**
   - الإيرادات → −COGS → =إجمالي الربح → −تشغيلي → −هدر → =صافي الربح
   - كل بند شريط نسبي ملوّن

3. **الميزانية العمومية:**
   - ٣ بطاقات: أصول / خصوم / حقوق ملكية
   - أشرطة تركيبية داخل كل بطاقة

4. **تصدير PDF:**
   - تقرير A4 احترافي: اسم المحل + تاريخ + قائمة الدخل + الميزانية + ٥ KPIs

5. **أزرار إجراءات:**
   - بجانب DSO: زر واتساب لإرسال تذكيرات تحصيل
   - زر خروج (تِراكوتّا) للعودة للوضع اليومي

**رحلة المستخدم النموذجية:**
1. يضغط «المستثمر» في الإعدادات → ينتقل التطبيق لخلفية بيضاء (٤٠٠ms)
2. يرى الأرقام الكبيرة فوراً — لا حاجة للتمرير لفهم الوضع
3. يطّلع على قائمة الدخل الشلالية — يفهم من أين يأتي الربح وأين يذهب
4. يضغط PDF → يحمّل تقريراً للمشاركة مع البنك
5. يضغط «خروج» → يعود للوضع اليومي

### التغييرات التقنية المطلوبة (مستقبلية)
- `avg_unit_cost` في items لتقييم دقيق للمخزون
- `wastage_records` لتحليل الهدر التفصيلي
- `fixed_assets` للأصول الثابتة (أجهزة، معدات)
- `theme-color` meta ينتقل من `#CC785C` إلى `#FFFFFF`

## التوصية المعمارية

1. **`mode` في TermsContext** يتحكم في: المصطلحات + التخطيط. هذا مقبول لـ٣ طبقات لكن إذا أُضيفت طبقة رابعة، يجب فصل `LayoutContext`.

2. **التبديل بين الطبقات** يحدث فوراً (لا reload) عبر تغيير `report_mode` في Dexie → `TermsContext` يعيد الرسم.

3. **Layer 2 و 3 يُبنيان فوق Layer 1** — لا يستبدلانه. المستخدم يمكنه العودة لليومي في أي وقت.
