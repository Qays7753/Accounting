# نظام التصميم — المرجع الموحّد (SOP · v3)

نظام مالي وإداري استثماري — **تطبيق هاتف فقط / PWA** (لا ديسكتوب حالياً).
المنطقة: الأردن · العملة: الدينار الأردني.
الاتجاه البصري: **تِراكوتّا دافئ (هوية كلود) + عاجي منفتح + محايدات دافئة + دلالات باردة مفصولة + نظام ارتفاع (عمق)**.

> مرجع مُلزِم قبل البناء وأثناءه وبعده. كل بند هنا معيار لا اقتراح.

**طبقات النظام:** Layer 1/2 = الوضع اليومي (هذا المستند §0–§12). **Layer 3 = وضع المستثمر (§13)** — لوحة تنفيذية فاتحة جريئة تُفتح بكبسة، لها استثناءات لونية موثّقة في §13 فقط.

### سجل التغييرات (v3)
- توحيد الهيدر عبر مكوّن `PageHeader` مشترك + `SegmentedControl` (§6.1).
- مقياس طباعة مسمّى بدل القيم العشوائية؛ **منع قاطع لـ`letter-spacing` السالب على العربية** (§5).
- طبقة التجربة: زرّ الرجوع/الخروج الأصلي، حارس الإرسال المزدوج، الضغط المطوّل للحذف، حبس التركيز، أمان لوحة المفاتيح (§8.5).
- نموذج تنقّل مسطّح (history) + خصوصية مالية (قفل/تعتيم) (§8.6).
- نظام السحب أزرق فولاذي (لا برتقالي)، والأساسي تِراكوتّا ثابت بلا مبدّل مستخدم.
- إضافة §13 وضع المستثمر (فاتح).

## 0. المبادئ الحاكمة

1. **هاتف أولاً وأخيراً:** عرض التصميم `390px` (المدى 360–430).
2. **أرقام فقط:** المبالغ أرقاماً **بدون** رمز عملة. صحيح: `12,400` — خطأ: `12,400 د.أ`.
3. **التواريخ أرقام:** `DD/MM/YYYY` (مثال `17/07/2026`). الوقت `HH:MM` (24 ساعة).
4. **انضباط لوني:** التِراكوتّا يملك اللِّين الدافئ وحده كهوية؛ الدلالات كلها باردة/خضراء/قرمزية لتنفصل عنه.
5. **الوضوح قبل الزخرفة:** لا تدرّجات صارخة، لا إيموجي، لا توهّج، لا حدود ملوّنة زخرفية.
6. **عمق هادئ:** العمق عبر نظام الارتفاع والتباين — لا عبر ظلال ثقيلة.
7. **اللون ليس الدليل الوحيد:** كل حالة مالية = لون + إشارة (`+`/`−`) + أيقونة (لعمى الألوان).
8. **لمسة آمنة:** أصغر هدف لمس `44×44px`.

## 5. الطباعة

الخط: **IBM Plex Sans Arabic** (مع `preload` لمنع وميض الخط الاحتياطي). الأرقام: **IBM Plex Mono** (`tnum`). أصغر مقاس `12px`.

**مقياس مسمّى (توكِنات — لا قيم عشوائية `text-[30px]`):**

|---|---|---|
| `title` | عنوان الشاشة الكبير | 28 / 700 |
| `title-sm` | العنوان المنكمش في الشريط | 20 / 700 |
| `section` | عنوان قسم | 17 / 600 |
| `card-title` | عنوان بطاقة | 15 / 600 |

## 6. بنية الشاشة

### 6.1 الهيدر الموحّد (مكوّن مشترك — إلزامي)
كل الشاشات تستخدم مكوّناً واحداً `PageHeader` — **ممنوع كتابة `<header>` يدوياً في أي صفحة**.
- **شريط علوي ملتصق (sticky) 56px** موحّد في كل الصفحات: عنوان + إجراء أو اثنان (أيقونات). حشو أفقي موحّد `px-4` (16). يحترم `safe-area-inset-top`.
- **العنوان الكبير المنكمش:** يبدأ `title` (28/700) في صفّ ثانوي، ينكمش إلى `title-sm` (20/700) داخل الشريط الـ56 عند التمرير.
- **الصفّ الثانوي (subheader) يمشي مع التمرير** (لا يلتصق): يحمل العنوان الكبير/بطاقة KPI/الـ`SegmentedControl`. الملتصق هو الشريط الـ56 فقط ليبقى الكروم ≤30%.
- **ظل التمرير:** الشريط مسطّح عند القمّة، ويظهر ظل مستوى ١ عند مرور المحتوى تحته فقط (`IntersectionObserver`).
- **البحث كأيقونة تتوسّع:** لا مربع بحث كامل ثابت — أيقونة في الشريط تتوسّع لحقل عند الضغط وتعود عند الإفراغ (توفير المساحة الرأسية).
- **`SegmentedControl` مكوّن واحد مشترك** (لا نسخ لكل صفحة).

### المنطقة العلوية
- شريط النظام ملك النظام؛ استخدم `safe-area-inset-top` ولوّن `theme-color` (يتماشى مع لون الهيدر/الوضع).
- شريط علوي فاتح (سطح/عاجي) بنص `#1F1E1D` — **ليس تِراكوتّا ممتداً**.
- إجراء واحد أو اثنان كحد أقصى + رجوع عند الحاجة.

### المنطقة السفلية
- تنقّل سفلي: 3–5 وجهات **رئيسية فقط** (لا إجراءات)، النشط بلون + أيقونة ممتلئة.
- الإجراء الأساسي: FAB أو شريط سفلي بزر أساسي في النماذج.
- شريط الإجراء السفلي يرتفع فوق لوحة المفاتيح ويحترم المنطقة الآمنة.

### التمرير مقابل شاشة واحدة
- **التمرير هو المعيار.** الكروم ثابت والمحتوى يمرّر بينه.
- أول مشهد (فوق الطيّة) يُظهر أهم معلومة + الإجراء الأساسي بلا تمرير.
- بلا تمرير للمهام المركّزة فقط: PIN/تسجيل، تأكيد إجراء واحد، البيع السريع/لوحة الأرقام.
- ممنوع: تمرير أفقي للمحتوى الرئيسي (فقط الشرائح)، وتمرير رأسي متداخل متصارع.
- سحب-للتحديث، والعودة لأعلى عند إعادة نقر التبويب.

## 8. الحالات والحركة وإمكانية الوصول

### 8.1 حالات كل شاشة (إلزامية)
- **فارغة:** أيقونة + جملة + زر إجراء.
- **تحميل:** هياكل skeleton لا سبينر.
- **خطأ/تحقّق:** رسالة أسفل الحقل، تأكيد للحذف.
- **نجاح · أول استخدام (حساب فارغ) · دون إنترنت.**

### 8.2 الحركة والتغذية الراجعة
- 150–300ms، easing موحّد (دخول ease-out، خروج ease-in).
- ضغط الزر 0.97، Toast بعد كل إجراء، اهتزاز خفيف، عدّ تصاعدي للأرقام، **بلا layout shift**.
- blur بندرة (شريط علوي/سفلي فوق المحتوى).

### 8.3 إمكانية الوصول
- تباين WCAG AA للنصوص.
- اللون ليس الدليل الوحيد (إشارة + أيقونة).
- حالات تركيز واضحة، دعم تكبير الخط، حد أدنى 12px.

### 8.4 الحالات الحديّة للبيانات
أسماء طويلة (`line-clamp` لا قصّ أعمى منتصف الكلمة)، أرقام ضخمة/صفر/سالبة، عنصر واحد مقابل مئات، قوائم طويلة (ترقيم/تحميل تدريجي)، تجاوز النص للحاوية. **تمييز «لا نتائج بحث» عن «لا بيانات».**

### 8.5 التفاعل وسلامة البيانات (طبقة التجربة — إلزامي)
- **زرّ الرجوع الأصلي:** كل ورقة/مودال/بحث موسّع يسجّل `history.pushState` عند الفتح؛ زرّ الرجوع (Android/إيماءة) يُغلق **آخر طبقة مفتوحة فقط**. مسار إغلاق واحد عبر `history.back()` (بلا سباق `setTimeout`).
- **الخروج:** من تبويب فرعي → الرجوع يعيد للرئيسية؛ من الرئيسية → «اضغط مرّة أخرى للخروج» خلال ٢ث.
- **منع الإرسال المزدوج (مالي):** كل زرّ حفظ/بيع/تسوية يُعطّل + سبينر أثناء الإرسال (حارس `submitting`) — لا حركة مالية مكرّرة.
- **الحذف:** بضغط مطوّل أو تأكيد صريح — **ممنوع الحذف بالنقر المزدوج**؛ حذف ناعم + نافذة تراجع للإجراءات المالية.
- **لوحة المفاتيح:** زرّ الإجراء يرتفع فوقها عبر `visualViewport` — **بلا `scrollIntoView`**.
- **حبس التركيز:** الأوراق `role="dialog"` + `aria-modal` + حبس Tab داخلها + إعادة التركيز للزر المُطلِق عند الإغلاق.
- **إدخال المبالغ:** `inputMode="decimal"`، تحقّق من الصفر/السالب/الضخم، بلا رمز عملة.

### 8.6 التنقّل والخصوصية
- **history مسطّح:** تبديل تبويبات الشريط السفلي بـ`replace` لا `push` — لا يتراكم المكدّس ولا يرجع عبر تبويبات مزارة.
- **خصوصية مالية:** تعتيم/إخفاء الشاشة في مبدّل التطبيقات، وقفل تلقائي (PIN) عند العودة من الخلفية بعد مهلة.
- **دون إنترنت:** مؤشّر يظهر/يختفي مع الاتصال (PWA).
- **الأساسي هوية ثابتة:** لا مبدّل لون للمستخدم؛ التخصيص (إن وُجد) وظيفي (أمان/عرض/بيانات/إشعارات) لا لوني.

## 9. RTL والمحتوى

- عكس صحيح للأيقونات الاتجاهية، محاذاة للنهاية (يمين).
- أرقام/تواريخ لاتينية داخل النص العربي، بمحاذاة جدولية `tnum`.
- صياغة أزرار بأفعال، صيغ جمع صحيحة، نبرة عربية طبيعية.
- طريقة واحدة لكل مهمة، وإعادة استخدام المكوّنات، وأسلوب أيقونات واحد.

---

## 10. الرسوم البيانية (للمستقبل)

- ألوان من اللوحة الدلالية لا قوس قزح؛ فئات مرتّبة (تِراكوتّا، تركواز، فولاذي، ذهبي، أخضر) بحد ~5–6 متمايزة، آمنة لعمى الألوان ومتمايزة بالرمادي.
- دخل أخضر، صرف قرمزي، صافٍ بلون الحبر/الهوية.
- قلّة الحبر: شبكة فاتحة `#EAE6DC`، محاور رفيعة، تسمية مباشرة، بلا حدود ثقيلة، بلا 3D.
- أرقام Mono/tnum، تفاعل باللمس (Tooltip بالنقر).
- الشكل حسب البيانات: خطي للاتجاه، أعمدة للمقارنة، دائري مجوّف للتركيب (≤5 شرائح).
- للرسوم حالات فارغة/تحميل، وارتفاع وحشو موحّدان.

---

## 11. صيغة الأرقام والتواريخ

- المبالغ: أرقام فقط، فاصل آلاف `,`، بلا رمز عملة، خط Mono.
- الاتجاه: داخل `+` / خارج `−` (علامة الطرح `−`) بلون الحالة + أيقونة.
- التاريخ `DD/MM/YYYY`، الوقت `HH:MM` (24)، محاذاة الأرقام للنهاية بـ`tnum`.

---

## 12. افعل / لا تفعل

**افعل:** أسطح فاتحة عاجية، تِراكوتّا كجُزُر صغيرة، عمق عبر الارتفاع، مسافات من الشبكة، حالات كاملة لكل شاشة، أيقونة + إشارة مع اللون.

**لا تفعل:** تِراكوتّا كسطح داكن ممتد · رمادي مزرق مع العاجي · تدرّجات/توهّج · حدّ وظل معاً · إيموجي · رموز عملة · أرقام حشو · الاعتماد على اللون وحده · تمرير أفقي للمحتوى الرئيسي · `letter-spacing` سالب على العربية · الحذف بالنقر المزدوج · مبدّل لون للمستخدم.

> **استثناء موثّق:** المنع أعلاه («تِراكوتّا سطح ممتد») يخصّ الوضع اليومي (Layer 1/2). في **وضع المستثمر (§13)** يُسمح بقلب استخدام اللون (تِراكوتّا كسطح مهيمن) — حصراً داخل تلك الطبقة.

## 13. وضع المستثمر (Layer 3) — لوحة تنفيذية فاتحة

لوحة تحكم تنفيذية تُفتح بكبسة وتُغلق. الإحساس المقصود: **«تقرير سنوي مطبوع / محطّة مالية فاتحة»** — أبيض أنصع + كتل تِراكوتّا جريئة + أرقام Mono ضخمة. **ليست وضعاً داكناً.** الجرأة من **قلب استخدام اللون** لا من تغيير الهوية.

### 13.1 الباليت (فاتح، مقلوب)
| الدور | الكود | ملاحظة |
|---|---|---|
| خلفية أساسية | `#FFFFFF` | أنصع من عاجي اليومي |
| خلفية أقسام | `#FBFAF7` | أنقى/أبرد قليلاً للإحساس التنفيذي |
| **بطاقة البطل (سطح تِراكوتّا)** | `#CC785C` / الأعمق `#B4613F` | نص عاجي `#FAF9F5` — **الاستثناء الجريء** |
| جزيرة الرقم الأهم | `#2A2521` | حبر دافئ داكن + أرقام بيضاء — جزيرة واحدة فقط |
| حدّ شعري | `#EAE6DC` | — |
| فاصل | `#DAD5C8` | — |
| نص رئيسي / ثانوي | `#1F1E1D` / `#6E6A60` | — |
| ربح (معمّق للجرأة) | `#1E7A4D` | على أبيض |
| خسارة | `#B42318` | على أبيض |
| عائد ذهبي | `#B08532` | نادر |

- التِراكوتّا هي **الجسر البصري** بين الوضعين — تبقى ظاهرة في كليهما فيتعرّف المستخدم على «نفس التطبيق».
- كل الدلالات معايرة لتعبر WCAG AA على أبيض.

### 13.2 الـ10 مؤشرات — هرم لا شبكة مسطّحة
- **صفّ بطل:** أهمّ ٢ (Runway + صافي الهامش) كبطاقتين كبيرتين، رقم Mono ضخم (34–40) + اتجاه سهم.
- **شبكة 2×2** لباقي الـ٨، مجمّعة تحت عناوين أقسام: **الربحية / الكفاءة / السيولة** (التجميع يقلّل الحمل الإدراكي).
- كل مؤشّر = رقم بارز واحد + إشارة/لون واحد. لا sparkline لكل بطاقة (يُقصر على البطل).

### 13.3 قائمة الدخل — شلّال عمودي (لا جدول)
صفوف متتابعة، كل بند شريط أفقي بطول نسبي من الإيرادات: الإيرادات (شريط تِراكوتّا كامل) → −COGS → =إجمالي الربح → −تشغيلي → −هدر → **=صافي الربح** (بطاقة بطل مميّزة). التسمية يمين، الرقم يسار Mono، الشريط التناسبي تحته.

### 13.4 الميزانية — ٣ بطاقات مكدّسة (لا جدول)
أصول / خصوم / حقوق ملكية. كل بطاقة: عنوان + رقم Mono كبير + شريط تركيبي ملوّن (كاش/مخزون/ذمم كأجزاء). معادلة `الأصول = الخصوم + الملكية` تظهر بصرياً بتساوي الأطوال.

### 13.5 الطباعة (إحساس Bloomberg-light)
- كل الأرقام `IBM Plex Mono` + `tabular-nums`، معزولة LTR داخل RTL (`unicode-bidi: isolate`).
- تباين حادّ: أرقام البطل 34–40/700، تسميات مصغّرة 11–12 (tracking خفيف لاتيني فقط). اللون = الدلالة، الحجم = الأولوية.

### 13.6 الانتقال البصري
- انتقال ٤٠٠–٥٠٠ms: الخلفية من عاجي `#FAF9F5` → أبيض ناصع، وكتل التِراكوتّا «تتمدّد» بحركة stagger خفيفة (شعور فتح غرفة الإدارة).
- يحترم `prefers-reduced-motion` (تلاشٍ فقط)، و`theme-color` ينتقل بالتزامن.
- زرّ الدخول/الخروج تِراكوتّا في الحالتين.
